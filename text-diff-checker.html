How to Use the Text Diff Checker

A Quick Look Under the Hood

Our tool uses the classic Longest Common Subsequence (LCS) algorithm. In simple terms, it finds the longest sequence of lines that appear in the same order in both texts. Everything else is flagged as added or removed. This is the same approach used by the Unix `diff` command and many version control systems.

The algorithm runs entirely in JavaScript inside your browser. That means no server round‑trips, no waiting, and complete privacy. For texts up to a few thousand lines, it's nearly instant. For very large files, your device's memory and CPU will be the only limiting factors.

Tips for Getting the Most Out of the Diff Checker

  • Keep line breaks consistent: The diff works line by line. If one version has different wrapping (e.g., hard returns vs. soft wrapping), the diff may show more changes than expected. Use a text editor to normalize line endings if needed.
  • Compare similar formats: For best results, paste plain text. If you compare heavily formatted text (like HTML), the diff will show markup changes along with content changes.
  • Use the Swap button: If you accidentally reversed the original and modified texts, just click "Swap Texts" to flip them without re‑pasting.
  • Save diff reports: Download the diff as a .txt file to keep a record of changes for future reference or to share with others.
